# Requirement Specification: Smart Pharmacy Web POS

ระบบบริหารจัดการร้านยาอัจฉริยะที่เน้นความถูกต้องตามกฎหมาย อย. และความปลอดภัยของผู้ป่วย

---

## 1. User Roles & Access Control (ระบบจัดการสิทธิ์)

ระบบรองรับการแบ่งสิทธิ์ผู้ใช้งาน (RBAC) เพื่อความปลอดภัยของข้อมูลยาควบคุม — JWT Authentication ผ่าน UM API ภายนอก (Single-tenant system)

| Role | สิทธิ์การใช้งาน |
|------|----------------|
| **SUPER** | สิทธิ์ทุกอย่าง รวมถึงจัดการผู้ใช้, ตั้งค่าระบบ, ดูรายงาน |
| **ADMIN** | จัดการตั้งค่าระบบ, รับสินค้าเข้าสต็อก, ดูรายงาน ข.ย., จัดการผู้ใช้ |
| **USER** | ทำรายการขาย (POS), ดู Dashboard, จัดการสินค้า/ผู้ป่วย |

- **User Management Page** — CRUD ผู้ใช้ผ่าน UM API, เปลี่ยน Role/Status/Password (SUPER, ADMIN เท่านั้น)
- **Profile Page** — ดู/แก้ไขโปรไฟล์ส่วนตัว
- **Sidebar เมนู** — แสดงเมนูตาม Role และ Feature Flag อัตโนมัติ
- **Authentication** — JWT token ตรวจสอบ clientId เพื่อยืนยันสิทธิ์การเข้าถึง API (ไม่ใช้สำหรับ filter ข้อมูล)

## 2. Product & Inventory Management (การจัดการคลังยา)

หัวใจสำคัญคือการคุม Lot Number และ Expiry Date

### 2.1 Product Management (จัดการสินค้า)

- **Product CRUD** — เพิ่ม/แก้ไข/ลบ/ดูรายละเอียดสินค้า พร้อม Soft Delete
- **Product Detail Page** — หน้ารายละเอียดสินค้าแบบเต็ม (`/inventory/[id]`)
- **CSV Bulk Import** — นำเข้าสินค้าจากไฟล์ CSV พร้อม preview, validation, progress bar
- **Report Types** — ผูกสินค้ากับรายงาน ข.ย. ที่เกี่ยวข้อง (เลือกได้หลายประเภท) - ระบบจะรวมสินค้าในรายงานตามที่กำหนดใน `reportTypes` และใช้สำหรับตรวจสอบการขายยาควบคุม
- **Product Fields** — barcode, ชื่อการค้า, ชื่อสามัญ, หมวดหมู่, ขนาดยา, หน่วย, ราคาทุน, ราคาขาย, min stock, ผลข้างเคียง, ข้อห้ามใช้, เงื่อนไขการเก็บ, drug interactions

### 2.2 Batch Tracking (คลังสินค้า)

- **Batch Management Page** — หน้าดูรายการ Lot ทั้งหมด (`/batches`) พร้อม search, pagination, เรียงตาม FEFO
- **Batch CRUD** — สร้าง/แก้ไข/ลบ Lot พร้อม Lot Number, วันหมดอายุ, จำนวน, ราคาทุน, ผู้จำหน่าย
- **FEFO Logic** — ระบบตัดสต็อกจาก Lot ที่ "หมดอายุก่อน" เสมอ (First Expired, First Out)

### 2.3 Goods Receipt (รับสินค้าเข้าสต็อก)

- **Multi-Item Receiving** — รับสินค้าหลายรายการในใบรับเดียว (Goods Receipt)
- **Receipt Number** — ระบบออกเลขที่ใบรับสินค้าอัตโนมัติ (GR-YYYYMMDD-XXXX)
- **Supplier & Notes** — บันทึกแหล่งที่ซื้อและหมายเหตุ
- **Pending Items** — เพิ่มรายการทีละตัว แสดง preview ก่อนบันทึกทั้งหมดพร้อมกัน
- **Atomic Validation** — ตรวจสอบทุกรายการก่อนบันทึก ป้องกันข้อมูลไม่สมบูรณ์
- **History** — ดูประวัติการรับสินค้าย้อนหลัง พร้อม pagination
- **Role Restriction** — เฉพาะ SUPER/ADMIN เท่านั้นที่เข้าถึงได้

### 2.4 Multi-Unit Management (การจัดการหน่วยนับที่ซับซ้อน)

ระบบรองรับการรับเข้าสต็อกในหน่วยใหญ่ (เช่น กล่อง) แต่ขายในหน่วยย่อย (เช่น แผง, เม็ด) ได้โดยอัตโนมัติ

- **UOM Conversion (แปลงหน่วยนับ):**
    - ผลิตภัณฑ์แต่ละตัวมี "หน่วยฐาน" (เช่น เม็ด) และสามารถกำหนดหน่วยนับเพิ่มเติมได้หลายระดับ
    - ตัวอย่าง: 1 แผง = 10 เม็ด, 1 กล่อง = 10 แผง = 100 เม็ด
    - สต็อกทั้งหมดเก็บในหน่วยฐาน — ระบบแปลงอัตโนมัติเมื่อรับเข้าหรือขายในหน่วยอื่น
    - แต่ละหน่วยมีราคาทุนและราคาขายแยกอิสระ
- **Barcode Mapping (ผูกบาร์โค้ดหลายชุด):**
    - สินค้าตัวเดียวสามารถมีบาร์โค้ดได้หลายชุด (เช่น บาร์โค้ดกล่อง vs บาร์โค้ดแผง)
    - เมื่อสแกนบาร์โค้ดหน่วยใด ระบบจะเลือกหน่วยนั้นให้อัตโนมัติ
    - ค้นหาสินค้าด้วยบาร์โค้ดของหน่วยใดก็ได้
- **Receiving (การรับเข้าสต็อก):**
    - สามารถเลือกหน่วยรับเข้า (เช่น กล่อง) ระบบจะแปลงเป็นหน่วยฐานในสต็อกอัตโนมัติ
    - แสดงจำนวนหน่วยฐานที่จะเข้าสต็อกให้เห็นก่อนบันทึก
- **POS (หน้าขาย):**
    - เมื่อเลือกสินค้าที่มีหลายหน่วย ระบบแสดง popup ให้เลือกหน่วยขาย พร้อมราคาแต่ละหน่วย
    - สินค้าตัวเดียวกันสามารถอยู่ในตะกร้าพร้อมกันได้หลายหน่วย (เช่น 2 กล่อง + 3 แผง)

### 2.5 Inventory Alerts

- **Low Stock** — แจ้งเตือนบน Dashboard เมื่อสต็อกต่ำกว่า Min Stock ที่กำหนด
- **Expiring Soon** — แจ้งเตือนสินค้าใกล้หมดอายุ (ภายใน 6 เดือน)

## 3. Point of Sale (POS) & Clinical Support

หน้าจอขายยาที่มากกว่าแค่คิดเงิน

- **Smart Search** — ค้นหาด้วยชื่อการค้า (Trade Name), ชื่อสามัญ (Generic Name), หรือ Barcode (รวมบาร์โค้ดหน่วยนับเพิ่มเติม)
- **Multi-Cart Tabs** — รองรับ 6 ตะกร้าพร้อมกัน สลับไปมาได้ทันที สำหรับรับลูกค้าหลายรายพร้อมกัน
- **Unit Selector** — สินค้าที่มีหลายหน่วยนับ จะแสดง popup ให้เลือกหน่วยขาย หรือสแกนบาร์โค้ดหน่วยใดก็เลือกอัตโนมัติ
- **Drug Interaction Alert** — แจ้งเตือนทันทีหากมียาในตะกร้าที่มีปฏิกิริยาต่อกัน
- **Allergy Check** — หากผูกผู้ป่วย แล้วพบว่าแพ้ยาที่กำลังจะขาย ระบบแจ้งเตือน Block ทันที
- **Controlled Drug Compliance** — ยาควบคุม (DANGEROUS, CONTROLLED, PSYCHO, NARCOTIC) บังคับบันทึก:
    - ชื่อเภสัชกรผู้จ่ายยา (pharmacistName)
    - ชื่อผู้สั่งจ่าย (prescriberName)
    - ชื่อผู้ซื้อ (buyerName)
    - เลขบัตรประชาชนผู้ซื้อ (buyerIdCard)
- **Prescription Label** — พิมพ์ฉลากยาอัตโนมัติ (ชื่อยา, ชื่อสามัญ, วิธีใช้, ผลข้างเคียง) รองรับขนาด 8×5 cm และ 5×3 cm
- **Multiple Payments** — รองรับเงินสด (CASH), โอนเงิน (TRANSFER), บัตรเครดิต (CREDIT_CARD) พร้อมคำนวณเงินทอน
- **Patient Linking** — เลือกผู้ป่วยจากระบบ CRM เพื่อตรวจสอบแพ้ยาและบันทึกประวัติการจ่ายยา
- **FEFO Deduction** — ระบบตัดสต็อกจาก Lot ที่หมดอายุก่อนอัตโนมัติ

## 4. Regulatory Compliance (ระบบรายงาน ข.ย.)

ฟีเจอร์บังคับสำหรับร้านยาในประเทศไทย — รองรับ 5 รายงาน:

| รายงาน | ชื่อเต็ม | ข้อมูลที่บันทึก |
|--------|----------|----------------|
| **ข.ย. 9** | บัญชีการซื้อยา | ลำดับที่,วัน/เดือน/ปี ที่ซื้อ,ชื่อและที่อยู่ของผู้ขาย,ชื่อยา,เลขที่หรืออักษรของครั้งที่ผลิต (Lot No.),จำนวน/ปริมาณที่ซื้อ,ลายมือชื่อผู้มีหน้าที่ปฏิบัติการ,หมายเหตุ |
| **ข.ย. 10** | บัญชีการขายยาควบคุมพิเศษ | ลำดับที่,วัน/เดือน/ปี ที่ขาย,ชื่อยา,เลขที่หรืออักษรของครั้งที่ผลิต (Lot No.),จำนวน/ปริมาณที่ขาย,ชื่อและสกุลของผู้ซื้อ,ลายมือชื่อผู้มีหน้าที่ปฏิบัติการ,หมายเหตุ |
| **ข.ย. 11** | บัญชีการขายยาอันตรายที่เลขาธิการ อย. กำหนด | ลำดับที่,วัน/เดือน/ปี ที่ขาย,ชื่อยา,เลขที่หรืออักษรของครั้งที่ผลิต (Lot No.),จำนวน/ปริมาณที่ขาย,ชื่อและสกุลของผู้ซื้อ,ลายมือชื่อผู้มีหน้าที่ปฏิบัติการ,หมายเหตุ |
| **ข.ย. 12** | บัญชีการขายยาตามใบสั่งแพทย์ | ลำดับที่,วัน/เดือน/ปี ที่ขาย,ชื่อยา,เลขที่หรืออักษรของครั้งที่ผลิต (Lot No.),จำนวน/ปริมาณที่ขาย,ชื่อ-สกุลผู้ใช้ยา,อายุผู้ใช้ยา,ที่อยู่ผู้ใช้ยา,ชื่อและสกุลผู้สั่งยา,สถานที่ทำงานของผู้สั่งยา,ลายมือชื่อผู้มีหน้าที่ปฏิบัติการ,หมายเหตุ |
| **ข.ย. 13** | รายงานการขายยาตามที่เลขาธิการ อย. กำหนด (ขายส่งยา) | ลำดับที่,วัน/เดือน/ปี ที่ขาย,ชื่อยา,เลขทะเบียนตำรับยา,เลขที่หรืออักษรของครั้งที่ผลิต (Lot No.),จำนวน/ปริมาณที่ขาย,ชื่อผู้ซื้อหรือชื่อสถานพยาบาล/ร้านยา,ที่อยู่ผู้ซื้อ,เลขที่ใบอนุญาตผู้ซื้อ,หมายเหตุ |

- **Date Range Filter** — เลือกช่วงวันที่ที่ต้องการออกรายงาน
- **Preview** — ดูตัวอย่างข้อมูลก่อน export
- **Export** — ส่งออกได้ทั้ง CSV และ PDF (client-side jsPDF + autotable)
- **Pharmacist Log** — บันทึกชื่อเภสัชกรผู้ปฏิบัติหน้าที่ในทุกรายการขายยาควบคุม
- **Role Restriction** — เฉพาะ SUPER/ADMIN เท่านั้นที่เข้าถึงรายงานได้

## 5. Patient Profile & CRM (ประวัติผู้ป่วย)

ฟีเจอร์นี้สามารถเปิด/ปิดได้จากหน้า Settings (Feature Toggle)

- **Patient CRUD** — เพิ่ม/แก้ไข/ลบ/ค้นหาผู้ป่วย พร้อม pagination
- **Patient Detail Page** — หน้ารายละเอียดผู้ป่วยแบบเต็ม (`/patients/[id]`)
- **Medical History** — บันทึกโรคประจำตัว (Chronic Diseases), ประวัติการแพ้ยา (Allergies พร้อม Severity: MILD/MODERATE/SEVERE)
- **Dispensing History** — ดูประวัติการจ่ายยาย้อนหลังจากระบบ POS
- **PDPA Compliance** — ระบบบันทึกความยินยอม (Consent) และวันที่ยินยอมในการเก็บข้อมูลสุขภาพ
- **Refill Reminder** — ระบบคำนวณวันยาหมดอัตโนมัติ (default 30 วัน) และแจ้งเตือนบน Dashboard
- **Patient Fields** — เลขบัตรประชาชน, ชื่อ-นามสกุล, โทรศัพท์, อีเมล, วันเกิด, เพศ, ที่อยู่, หมายเหตุ

## 6. Dashboard & Analytics (การวิเคราะห์)

| Feature | Description |
|---------|-------------|
| **Today's Stats** | ยอดขายวันนี้, จำนวนสินค้าในคลัง, จำนวนยาใกล้หมดอายุ |
| **Sales Chart** | กราฟยอดขาย toggle ได้ระหว่างรายวัน (7 วัน) และรายเดือน (12 เดือน) |
| **Gross Margin** | กำไรขั้นต้นประจำเดือน (รายได้ vs ต้นทุน) พร้อม margin % |
| **ABC Analysis** | จัดกลุ่มสินค้าตามยอดขาย 6 เดือน — A (ขายดี), B (ปานกลาง), C (ขายน้อย/สต็อกจม) |
| **Dead Stock** | รายงานสินค้าไม่เคลื่อนไหวเกิน 90 วัน |
| **Expiring Items** | รายการยาใกล้หมดอายุภายใน 6 เดือน |
| **Low Stock** | รายการสินค้าที่สต็อกต่ำกว่า Min Stock |
| **Refill Reminders** | รายชื่อผู้ป่วยที่ถึงกำหนดซื้อยาซ้ำ พร้อมวันที่คาดว่าหมด |

## 8. Technical Requirements (ข้อกำหนดทางเทคนิค)

### Tech Stack

| Layer | Technology |
|-------|-----------|
| **Backend** | Go 1.25+, Gin framework, MongoDB, Redis (optional) |
| **Frontend** | Next.js 16 (App Router, Static Export), TypeScript, TailwindCSS, shadcn/ui, Zustand |
| **Auth** | JWT via UM API (external), RBAC middleware |
| **PDF** | jsPDF + jspdf-autotable (client-side) |

### Non-Functional

- **Platform** — Web Responsive (PC, Tablet, iPad)
- **Security:**
    - HTTPS / SSL Certificate
    - JWT Token Authentication
    - Role-Based Access Control (RBAC) middleware
    - Data Encryption สำหรับข้อมูลสุขภาพ
    - Daily Backup บน Cloud Storage
- **Hardware Integration** — รองรับเครื่องยิงบาร์โค้ด (USB HID), เครื่องพิมพ์ใบเสร็จ (Thermal), เครื่องพิมพ์ฉลากยา

## 9. สรุป Workflow การทำงาน

### 9.1 รับสินค้าเข้าสต็อก
1. เลือกสินค้า → กรอก Lot Number + วันหมดอายุ + จำนวน + ราคาทุน
2. (ถ้ามีหลายหน่วย) เลือกหน่วยรับเข้า → ระบบแสดงจำนวนหน่วยฐานที่จะเข้าสต็อก
3. เพิ่มรายการทีละตัว → review ทั้งหมด → บันทึกพร้อมกัน
4. ระบบสร้าง Goods Receipt + Batch อัตโนมัติ

### 9.2 ขายสินค้า (POS)
1. สแกนบาร์โค้ด หรือ ค้นหาด้วยชื่อ
2. (ถ้ามีหลายหน่วย) เลือกหน่วยขาย หรือสแกนบาร์โค้ดหน่วยที่ต้องการ
3. ระบบเช็ค Drug Interaction + Allergy Check อัตโนมัติ
4. (ยาควบคุม) บันทึกชื่อเภสัชกร/ผู้ซื้อ/เลขบัตร
5. พิมพ์ฉลากยา (ถ้าต้องการ)
6. ชำระเงิน → ระบบตัดสต็อกตาม FEFO อัตโนมัติ
7. บันทึกลงรายงาน ข.ย. ที่เกี่ยวข้องอัตโนมัติ

### 9.3 รายงาน
1. เลือกประเภทรายงาน (ข.ย. 9–13) → กำหนดช่วงวันที่
2. Preview ข้อมูล → Export CSV หรือ PDF
